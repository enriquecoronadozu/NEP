Use Publish/Subscribe pattern

 

In this pattern

  • Messages are send asynchronously.
  • The publisher send messages without stopping its execution
  • The subscriber can follow it execution even if a new message do not arrives
  • The subscriber can execute a callback function each time a new message arrives (ROS re-usable)

Use Client-Server pattern

 
 

In this pattern

  • A client (node 1) sends a request for some data to a server (node 1)
  • The server waits until it receives a request of some client
  • The server process the request and send a response to the client
  • The client waits until it receives some response of the server
