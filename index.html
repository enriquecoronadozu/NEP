Main concepts Nodes Sockets Client/Server Publish/Subscribe

Nodes are simple, loosely coupled, independent and re-usable building blocks of robot applications.

They usually run in parallel in one or different machines and can be written in different programming languages.

Communication between nodes is often done using sockets.

A network socket is a inter-process communication method for sending or receiving data within a node on a computer network.

Socket address for endpoints are composed of a protocol, an IP address, and a port value.

An example of endpoint definition is: tcp://127.0.0.1:5000 where: IP indicates the protocol, 127.0.0.1 indicates the IP address, and 5000 indicates the port of the connection.

In the Request/Process/Reply pattern, also denoted as Client/Server, a process sends a request for some data and then waits for the response to this request.

This pattern is suitable for the cases where synchronization between two tasks is needed. However, if a reply never comes, a deadlock can occur (i.e., a read operation cannot be performed when a write operation is in progress, and vice versa).

In the Publish/Subscribe model the producers of messages, called publishers, send messages without the knowledge of what or if any receiver, called subscribers. Unlike typical Client/Server models, the communication is asynchronous.

The notion of topics is often used to identify socket endpoints and filter the information sent by publishers
